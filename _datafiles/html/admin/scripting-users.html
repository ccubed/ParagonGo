User Scripts

One global script that runs for every player, hooking key lifecycle and command events.

Overview

Unlike rooms, mobs, items, and the other script types — which attach a separate script to each object — there is a single, global user script shared by every player. It is loaded once into one cached VM and its event handlers are invoked for whichever player triggered the event.

Use it for cross-cutting per-player behavior that should apply everywhere: welcome/farewell messages, level-up rewards, custom death handling (such as a phoenix-style revival), global command interception, and bleed-out effects while downed.

Edit the script in the User Script editor. This page is reference documentation for the available event hooks.

Script Path

The user script lives at a fixed path under the data files directory. A JavaScript file (.js) is preferred; a Lua file (.lua) is used only when no JavaScript file exists.

User script _datafiles/scripts/user.js

Editing & Reloading

Save the script through the User Script editor or the REST API. Saving invalidates the cached VM, so changes take effect on the next event without a server restart. Saving an empty script deletes the file and disables all handlers.

REST API GET  /admin/api/v1/user/script
PUT  /admin/api/v1/user/script  (requires users.write)

Script State

Because there is one shared VM, global variables in the user script are shared across all players. For per-player data, use user.SetTempData() (session-scoped) or user.SetMiscCharacterData() (persisted) on the actor passed into each handler.

Event Hooks

Define any subset of the handlers below. Every handler receives the affected player as an ActorObject and their current RoomObject. Handlers tagged Can Abort can suppress the engine's default behavior by returning true.

onLogin(user ActorObject, room RoomObject)Info

Called when a player enters the world (spawns in). Useful for welcome messages, daily login rewards, or refreshing per-session state. The return value is ignored.

ArgumentTypeDescription
userActorObjectThe player logging in.
roomRoomObjectThe room the player spawned in.
function onLogin(user, room) {
    user.SendText('Welcome back, ' + user.GetCharacterName(false) + '!');
}
function onLogin(user, room)
    user:SendText("Welcome back, " .. user:GetCharacterName(false) .. "!")
end
onLogout(user ActorObject, room RoomObject)Info

Called when a player leaves the world (despawns). Fires before the engine's final cleanup so the actor is still resolvable. The room argument may be null in some disconnect scenarios — always null-check it before use. The return value is ignored.

ArgumentTypeDescription
userActorObjectThe player logging out.
roomRoomObjectThe room the player was in, or null.
function onLogout(user, room) {
    if (room) {
        room.SendText(user.GetCharacterName(false) + ' fades from view.', user.UserId());
    }
}
function onLogout(user, room)
    if room ~= nil then
        room:SendText(user:GetCharacterName(false) .. " fades from view.", user:UserId())
    end
end
onLevel(user ActorObject, room RoomObject, eventDetails LevelEventDetails)Info

Called when a player gains one or more levels. The eventDetails object describes the change, including the previous level so you can compute how many levels were gained or trigger milestone rewards. The return value is ignored.

ArgumentTypeDescription
userActorObjectThe player who leveled up.
roomRoomObjectThe room the player is in.
eventDetailsLevelEventDetailsLevel change details (see below).
eventDetails FieldTypeDescription
newLevelnumberThe player's new (current) level.
previousLevelnumberThe level the player was before this event.
levelsGainednumberHow many levels were gained at once.
trainingPointsnumberTraining points granted by this level change.
statPointsnumberStat points granted by this level change.
livesGainednumberExtra lives granted (permadeath configurations).
function onLevel(user, room, eventDetails) {
    user.SendText('You are now level ' + eventDetails.newLevel + '!');
    // Milestone reward at level 10
    if (eventDetails.previousLevel < 10 && eventDetails.newLevel >= 10) {
        user.AddGold(1000);
        user.SendText('A milestone bonus of 1000 gold has been awarded!');
    }
}
function onLevel(user, room, eventDetails)
    user:SendText("You are now level " .. eventDetails.newLevel .. "!")
    -- Milestone reward at level 10
    if eventDetails.previousLevel < 10 and eventDetails.newLevel >= 10 then
        user:AddGold(1000)
        user:SendText("A milestone bonus of 1000 gold has been awarded!")
    end
end
onCommand(cmd string, rest string, user ActorObject, room RoomObject)Can Abort

Called whenever a player types any command. This runs before mob and room onCommand handlers and before the built-in command itself. Return true to consume the command and halt all further processing.

Return value: true = consumed (no mob/room scripts, no built-in command runs). false or nothing = pass through.

This fires for every command from every player. Keep it fast and return false early for commands you do not handle. Admin/server commands are never routed through scripts.

ArgumentTypeDescription
cmdstringThe command word typed (e.g. look, north).
reststringEverything entered after the command word.
userActorObjectThe player who typed the command.
roomRoomObjectThe room the player is in.
function onCommand(cmd, rest, user, room) {
    if (cmd === 'salute') {
        room.SendText(user.GetCharacterName(false) + ' snaps a crisp salute.');
        return true; // consume; no built-in 'salute' needed
    }
    return false;
}
function onCommand(cmd, rest, user, room)
    if cmd == "salute" then
        room:SendText(user:GetCharacterName(false) .. " snaps a crisp salute.")
        return true -- consume; no built-in 'salute' needed
    end
    return false
end
onDying(user ActorObject, room RoomObject)Can Abort

Called when a player is downed — bleeding out but not yet fully dead. This fires again each round while the player remains downed and unhealed. Return true to suppress the default downed handling (the “you drop to the ground” message and nearby-mob reactions) for that round.

Return value: true = suppress default downed handling this round. false or nothing = default behavior runs.

Aborting does not heal the player — they keep bleeding out and this handler is called again next round. To stop the bleed-out, heal the player (e.g. user.AddHealth(...)) so they leave the downed state.

ArgumentTypeDescription
userActorObjectThe downed player.
roomRoomObjectThe room the player is in.
function onDying(user, room) {
    user.SendText('Your vision blurs as your strength bleeds away...');
    return true; // suppress the default downed messaging this round
}
function onDying(user, room)
    user:SendText("Your vision blurs as your strength bleeds away...")
    return true -- suppress the default downed messaging this round
end
onDie(user ActorObject, room RoomObject)Can Abort

Called when a player fully dies, before any death penalties are applied (XP loss, corpse creation, item drops, teleport to the death recovery room). Return true to abort the death entirely — no penalties run and the script is responsible for whatever happens instead, such as a phoenix-style revival.

Return value: true = abort death (no penalties, no corpse). false or nothing = normal death proceeds.

When you abort, the engine ensures the player is left alive (health is restored to at least 1) and clears pending damage so they do not immediately re-die in the same combat resolution. Do not synchronously trigger another death from inside this handler.

ArgumentTypeDescription
userActorObjectThe dying player.
roomRoomObjectThe room the player is in.
function onDie(user, room) {
    // Phoenix charm: revive once instead of dying
    if (user.GetMiscCharacterData('phoenix') === 1) {
        user.SetMiscCharacterData('phoenix', 0);
        user.SetHealth(user.GetHealthMax());
        room.SendText(user.GetCharacterName(false) + ' is reborn in a burst of flame!');
        user.SendText('The phoenix charm shatters and pulls you back from death!');
        return true; // abort the death
    }
    return false;
}
function onDie(user, room)
    -- Phoenix charm: revive once instead of dying
    if user:GetMiscCharacterData("phoenix") == 1 then
        user:SetMiscCharacterData("phoenix", 0)
        user:SetHealth(user:GetHealthMax())
        room:SendText(user:GetCharacterName(false) .. " is reborn in a burst of flame!")
        user:SendText("The phoenix charm shatters and pulls you back from death!")
        return true -- abort the death
    end
    return false
end
onLoad()Info

Called once when the global user script is first loaded into memory (and again after a save reloads it). Use it for one-time initialization of shared script state. It has an extended timeout for setup work and takes no arguments.

function onLoad() {
    // one-time setup of shared script state
}
function onLoad()
    -- one-time setup of shared script state
end
